## 概要

Claudeがユーザーに質問・確認・選択を求めるとき、ターミナルの `ask_user_input` の代わりにtsumugiのGUIフォームを使う。
定義リストのフォーム要素（チェックボックス・ラジオボタン・テキスト入力）でユーザーに回答してもらい、`query` で結果を取得する。

## いつ使うか

- ユーザーに選択肢から選んでもらうとき
- 複数の質問をまとめて聞きたいとき
- 選択肢＋自由入力（「その他」パターン）が必要なとき
- 確認チェックリストを提示するとき

## フォーム記法リファレンス

フォーム要素は**定義リストの `: ` 行内でのみ有効**。定義語（`<dt>`）が質問文、配下の `: ` 行がフォーム要素となる。

| 記法 | 要素 | 用途 |
|------|------|------|
| `[ ]` / `[x]` | チェックボックス | 複数選択 |
| `[R:""]` / `[R:"x"]` | ラジオボタン | 排他選択（同一グループ内で1つだけ） |
| `[T:""]` / `[T:"初期値"]` | テキスト入力 | 自由入力 |
| `[R:""][T:""]` | ラジオ＋テキスト | 「その他（自由入力）」パターン |

## 使用手順

### 1. フォームを表示

`launch` でフォーム付きMarkdownを新規ウィンドウに表示する。

```
launch(
    title="確認",
    body="## 質問タイトル\n\n質問内容\n: [R:\"\"] 選択肢1\n: [R:\"\"] 選択肢2\n: [R:\"\"][T:\"\"]"
)
```

### 2. ユーザーに回答を促す

ターミナルに「tsumugiのウィンドウで回答してください」と表示して待つ。
ユーザーが回答完了を伝えてくれるのを待つ。

### 3. 回答を取得

ユーザーが回答完了を伝えたら `query` で本文を取得する。

```
query(instance_id="<対象ID>", properties=["body"])
```

### 4. 回答をパース

取得したMarkdownから各フォーム要素の値を読み取る。

- チェックボックス: `[x]` が選択済み、`[ ]` が未選択
- ラジオボタン: `[R:"x"]` が選択された項目
- テキスト入力: `[T:"入力値"]` から値を取得

## フォーム設計パターン

### パターン1: 排他選択（ラジオボタン）

```markdown
## どの方針で進めますか？

方針を選択してください
: [R:""] 既存APIを拡張
: [R:""] 新規APIを作成
: [R:""] 外部サービスを利用
```

### パターン2: 排他選択＋その他（ラジオ＋テキスト）

```markdown
## 使用する言語

言語
: [R:""] Python
: [R:""] TypeScript
: [R:""] Rust
: [R:""][T:""]
```

### パターン3: 複数選択（チェックボックス）

```markdown
## デプロイ前の確認

確認事項
: [ ] テストが全て通っている
: [ ] マイグレーションの確認済み
: [ ] 環境変数の設定済み
```

### パターン4: テキスト入力

```markdown
## プロジェクト設定

プロジェクト情報
: [T:""] プロジェクト名
: [T:""] リポジトリURL
```

### パターン5: 複合フォーム（複数の質問をまとめて）

```markdown
## 実装方針の確認

アーキテクチャ
: [R:""] モノリス
: [R:""] マイクロサービス
: [R:""][T:""]

対象環境
: [ ] development
: [ ] staging
: [ ] production

補足事項
: [T:""] 補足があれば入力してください
```

## 注意事項

- フォーム要素は定義リスト内でのみ有効。通常の段落やリストに書いてもテキストとして表示される
- ラジオボタンは同一の定義語（`<dt>`）配下で自動的にグループ化される。異なる質問は異なる定義語にする
- `launch` が失敗した場合はtsumugiが起動していないため、ユーザーにtsumugiの起動を依頼する
- ユーザーが回答完了を伝えるまで `query` しない
